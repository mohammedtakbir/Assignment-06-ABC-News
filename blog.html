Frequently Asked Inrterview Questions

  • var declarations are globally scoped or function/locally scoped.The scope is global when a Var variable is declared outside a function.Var is function scoped when it is declared within a function. var variables can be re-declared and updated.
  • let is block scoped. So a variable declared in a block with let is only available for use within that block.let can be updated but not re-declared.
  • const declarations can only be accessed within the block they were declared.Every const declaration, therefore, must be initialized at the time of declaration. const cannot be updated and re-declared.

  • If the arrow function contains one expression, you can omit the curly braces and then expression will be implicitly return. In regular function, you have to use return keyword to return any value.If you don't return, function will be indefined.
  • In an object method, inside of arrow function this don't refers the owner object.But, inside of regular function this refers to the owner object.
  • We cannot have duplicate parameters in arrow function.But we can duplicate parameters in regular function.

  • map() creates a new array from calling a function for every array element. map() calls a function once for each element in an array. map() does not execute the function for empty elements.
  • The forEach() method executes a provided function once for each array element. The forEach() method does not create a new array based on the given array. Return value undefined.
  • The filter() method creates a new array filled with elements that pass a test provided by a function. The filter() method does not change the original array.
  • The find() method returns the value of the first element that passes a test. The find() method returns undefined if no elements are found.

  • Template literals are sometimes informally called template strings, because they are used most commonly for string interpolation (to create strings by doing substitution of placeholders).
  • Its allowing for multi-line strings, for string interpolation with embedded expressions, and for special constructs called tagged templates.
  • That's why we use Template literals.
